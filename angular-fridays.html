Two way data-binding

HTML is the template (expands vocabulary of HTML)

Declarative Syntax

jQuery


<div class="myDiv"></div>
$('.myDiv').click(function() {
    $('.myDiv').toggleClass('disabled');
});

Angular


<div ng-class=”{ disabled: !myDivIsEnabled }”
ng-click=”myDivIsEnabled = !myDivIsEnabled></div>

If myDiv is not enabling we know exactly where to look!

Directives

  • Encapsulate DOM logic
  • Where all the 'two-way binding' happens.
  • Almost everything you see in an Angular template that isn't plain HTML is a directive. ng-class, ng-repeat, ng-controller: ALL DIRECTIVES! Even the script tag is a special directive used by Angular to find embedded templates.

Why Angular?

Separation of Concerns

By splitting our code into templates, directives, controllers, and services, we make it easier to maintain and debug code over time.

Prevents bugs.

Community

  • Tons of modules.
  • Full support for Bootstrap.

Angular is great for integration with existing applications.

  • Co-exists with other patterns.
  • Wraps jQuery plugins just fine.
  • Scales well to many developers.

Pattern Library

BoomTown has created our own Pattern Library with components (directives) found on common pages to make feature development quicker. The developer just needs to provide data to get a component that has already been laid out for them
